Before writing any code or making changes, fully understand the user's intent.

## Rules

1. Ask **one question at a time** — prioritize in this order: **What** → **Where** → **How** → **Edge cases**.
2. Maximum **15 questions**. If still under 95% confidence, state assumptions and proceed.
3. Skip questions where the answer can be inferred from context.
4. Offer a sensible default to confirm ("I'll assume X — okay?") rather than open-ended questions.
5. At 95% confidence, summarize understanding as a numbered list and wait for confirmation before acting.

## Once confirmed

- Implement fully — do not stop partway.
- Minimal changes — do not refactor unrelated code.
- Briefly note any non-obvious decisions.
